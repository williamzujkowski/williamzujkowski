## How It Works

The animated SVG terminal is generated by [svg-terminal](https://github.com/williamzujkowski/svg-terminal). All content and configuration lives in `terminal.yml`.

**To update the terminal content:** Edit `terminal.yml` and push — `.github/workflows/update-terminal-svg.yml` regenerates both SVGs via the published `williamzujkowski/svg-terminal@v1` GitHub Action (no local build needed) and commits them. Also runs every 6h for the daily theme rotation + live data.

**To regenerate locally:**
```bash
# Install the published CLI (one-time)
npm install -g svg-terminal

# Animated + static (the static one is the prefers-reduced-motion fallback)
svg-terminal generate --config terminal.yml --output src/terminal.svg
svg-terminal generate --config terminal.yml --output src/terminal-static.svg --static
```

## Blocks Used

See `terminal.yml` for full config. Current blocks (9): `motd` (live weather), `neofetch` (persona), `npm-install` (playful global install), `build-badge` (status card), `github-stats` (live), `github-languages` (live language breakdown), `jumping-jack` (multi-line animation), `dad-joke` (signature), `goodbye` (exit). Theme is `random` (daily rotation across all 20), glow/scanlines off. The animated SVG uses a FIXED landscape viewport (`maxHeight: 600`, 1000×600 ≈ 5:3) so the session types in and **scrolls** like a real terminal (~26s loop); the reduced-motion static fallback expands to full content height (svg-terminal #129, needs ≥ v1.2.3) so nothing is clipped.
